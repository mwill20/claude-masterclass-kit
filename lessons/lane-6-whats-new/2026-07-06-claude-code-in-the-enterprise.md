## A. Index
Rolling Claude Code out beyond one laptop: managed settings and policy
files, permission modes as organizational policy, shared CLAUDE.md and
command conventions, and cost/usage visibility. The individual tool becomes
a fleet.

## B. Build
Write a one-page adoption memo for your own team: which permission mode is
default, which tools are allowlisted, where shared commands live, and the
one metric you would watch in month one. Pressure-test it by configuring a
repo to match the memo.
You're done when: a fresh session in that repo actually enforces the memo's
permission mode and allowlist.
