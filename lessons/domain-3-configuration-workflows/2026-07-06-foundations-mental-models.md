## A. Index
The mental model that makes Claude Code predictable: it is a senior pair
with amnesia. Everything it knows per session comes from CLAUDE.md, the
files it reads, and your messages. Good sessions are engineered context,
not clever prompts.

## B. Build
Write a CLAUDE.md for one of your real projects: what it is, the commands
that matter, the conventions to follow, the things never to touch. Start a
fresh session and ask Claude Code to describe the project and its rules.
You're done when: the description matches your intent without correction,
and one deliberately wrong instruction you planted ("tests live in /qa")
gets followed, proving the file is actually load-bearing.
